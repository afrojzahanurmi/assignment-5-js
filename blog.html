What is DOM?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a document, typically HTML or XML, as a tree of objects. Each element, attribute, and piece of text in the document is represented as a node in this tree.

What are nodes in the DOM? Can you name the different types of nodes?

1. Element Node: Represents an HTML or XML element. For example, tags are element nodes..
2. Text Node: Represents the text content within an element. For example, in, is a text node.
3. Attribute Node: Represents an attribute of an element, though attributes are typically accessed through their parent element rather than as standalone nodes.
4. Comment Node: Represents comments in the markup, such as "This is a comment "
5. Document Node: Represents the entire document. It serves as the root of the DOM tree and provides access to all other nodes.

What is event delegation in the context of the DOM?

Event delegation is a technique used in the DOM to handle events more efficiently by leveraging the concept of event bubbling. Instead of attaching event listeners to individual child elements, you attach a single event listener to a common ancestor element. This allows the event to be handled at a higher level in the DOM hierarchy, which can improve performance and simplify code.
